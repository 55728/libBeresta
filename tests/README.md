# Система тестирования libBeresta

Тестирование кода сделано максимально простым.

В CMakeLists.txt подготавливаются бинарные файлы для отдельных тестов,
генерирующие либо какой-то вывод (вроде номера версии библиотеки),
либо создающие с помощью разных частей библиотеки PDF-файл.

Затем бинарные файлы запускаются и вывод либо используется напрямую,
либо PDF-файл анализируется с помощью утилиты `qpdf`.

Сами тесты записаны в файлы `*.test`, они фактически собой представляют
комбинацию команд запуска и запись вывода утилит.

Файл `run.sh` используется при сборке и вызывается из Makefile для снятия
кавычек с аргументов вызова утилит. Если то же самое можно будет сделать
внутри CMake, его можно будет удалить.

Используются утилиты [clitest](https://github.com/aureliojargas/clitest) и [qpdf](http://qpdf.sourceforge.net).
